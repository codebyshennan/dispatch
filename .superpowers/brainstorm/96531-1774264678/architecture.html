Architecture: Demo Harness

Two new packages. IntelligencePanel stays completely unchanged. The mock ZAF client is the key seam.

What gets built

apps/demo (new)

Vite React app. Split layout: left input panel, right analysis panel. Provides a DemoClientProvider that mocks the ZAF client — client.request() becomes fetch(localhost:3001). Imports IntelligencePanel from apps/sidebar/src/panels/ directly.

DemoApp.tsx — split layout
DemoClientProvider.tsx — mock ZAF
InputPanel.tsx — chat history + textarea
apps/demo-server (new)

Hono Node.js server. On startup embeds 115 KB articles with Vectra (TypeScript, no native deps). Exposes the same API surface as sidebar-api Lambda. Runs the full pipeline in-process — no AWS, no DynamoDB.

POST /analyze — submit ticket
GET /context/:id — poll result
GET /mode — returns agent_assisted

The mock ZAF client (the key seam)

// DemoClientProvider.tsx — IntelligencePanel never knows it's not in Zendesk
const mockClient = {
  context: () => Promise.resolve({ ticketId: currentTicketId }),
  request: ({ url, type, data }) => {
    const actual = url.replace('{{setting.api_base_url}}', 'http://localhost:3001');
    return fetch(actual, { method: type, body: data }).then(r => r.json());
  },
  on: () => {}, off: () => {}, get: () => Promise.resolve({}), ...
}
