3 approaches to the offline demo

All use Vectra (pure TypeScript vector store — no native bindings, just npm install). Pick based on how much you want to reuse existing code vs build time.

A

Demo harness around existing sidebar ⭐ Recommended

Add a thin apps/demo Vite app to the monorepo. It wraps the existing IntelligencePanel with a mock ZAF client (swaps client.request() → fetch() to a local server). A small Hono server imports the classifier + response-generator Lambda logic directly as modules — no AWS needed.

Left panel
Ticket/question textarea
+ Analyze button
+ message history
Right panel
Existing IntelligencePanel
(unchanged component)
+ KB article cards
Stack: Vectra (embed on startup) · existing Lambda handlers as modules · Anthropic API · mock ZAF client
Run: node server.js && pnpm dev — two commands
Build time: ~3–4h
B

New standalone demo app (chat-first)

A purpose-built chat UI (new React app, clean slate). Left: chat bubbles with message history. Right: analysis card built from scratch to match IntelligencePanel visually. FAISS via faiss-node for embeddings. Doesn't reuse any existing UI components.

Stack: faiss-node (requires native build) · Express · new React components
Run: python -m pip install... or npm rebuild faiss-node first
Build time: ~6–8h + native dep setup risk
C

Single-page HTML (no build)

One demo.html file with inline JS. Open in browser, type a question, get a response. Calls Anthropic API directly. No vector store — stuffs all 115 article titles+summaries into context window. Quickest to run but limited retrieval quality and no real classification pipeline.

Stack: vanilla JS · Anthropic API (direct fetch)
Run: open demo.html
Build time: ~1h but no real RAG, no existing UI reuse
