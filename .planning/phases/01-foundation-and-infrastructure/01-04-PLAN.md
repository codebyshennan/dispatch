    4. `runEval()` currently calls `process.exit()` at the end — this is wrong for Lambda usage. The function needs to be refactored to return a result object instead of calling `process.exit()`. Move the exit-code logic into the CLI entrypoint only.

    Refactoring `runner.ts`:
    - Change `runEval()` return type from `Promise<void>` to `Promise<EvalOutput>` (return the `output` object instead of printing and exiting).
    - Remove the `process.stdout.write` and `process.exit()` calls from inside `runEval()`.
    - Move the printing (`process.stdout.write`) and exit logic into the CLI commander `.action()` handler in `runner.ts`.
    - The CLI action becomes: `const output = await runEval(opts); process.stdout.write(...); if (accuracyRaw < threshold) process.exit(1); process.exit(0);`

    Updated `index.ts` handler:
    ```ts
    import { Hono } from 'hono';
    import { handle } from 'hono/aws-lambda';
    import { runEval } from './runner.js';

    const app = new Hono();

    app.get('/health', (c) => c.json({ status: 'ok' }));

    app.post('/eval/run', async (c) => {
      const body = await c.req.json<{ prompt?: string; dataset?: string; threshold?: number }>();

      if (!body.prompt || !body.dataset) {
        return c.json({ error: 'prompt and dataset are required' }, 400);
      }

      const threshold = body.threshold ?? 85;

      try {
        const output = await runEval({
          promptName: body.prompt,
          datasetName: body.dataset,
          threshold,
        });
        const passed = parseFloat(output.summary.accuracy) >= threshold;
        return c.json({ ...output, passed }, passed ? 200 : 422);
      } catch (err) {
        const message = err instanceof Error ? err.message : String(err);
        return c.json({ error: message }, 500);
      }
    });

    export const handler = handle(app);
    ```

    Note on `runner.ts` changes:
    - `EvalOutput` interface must be exported from `runner.ts` so `index.ts` can use the return type.
    - The CLI commander `.action()` must now capture accuracy from the returned output to decide the exit code (accuracy is in `output.summary.accuracy` as a string like "85.0%").
    - Parse accuracy: `const accuracyRaw = parseFloat(output.summary.accuracy);`

    Files to modify: `lambdas/eval/src/runner.ts` (refactor runEval return type) AND `lambdas/eval/src/index.ts` (implement handler).
