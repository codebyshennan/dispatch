  <name>Task 1: Enable pgvector via CDK AwsCustomResource in MeridianStack</name>
  <files>infra/src/stacks/meridian-stack.ts</files>
  <action>
    After the `this.dbCluster` definition in `MeridianStack`, add a CDK `AwsCustomResource` that runs an RDS `executeStatement` or `modifyDBClusterParameterGroup` — but the correct approach for enabling a Postgres extension is a custom resource that runs SQL via the RDS Data API, or more simply: use `cr.AwsCustomResource` with `rds.executeStatement` to run `CREATE EXTENSION IF NOT EXISTS vector;` against the `meridian` database.

    However, the Aurora cluster is in a VPC with isolated subnets (in dev) and no public access, so direct SQL execution from CDK requires the RDS Data API to be enabled on the cluster or a Lambda-backed custom resource in the same VPC.

    The simpler and production-safe approach: add a `CfnDBClusterParameterGroup` or use a CDK `custom_resources.AwsCustomResource` targeting `RDS:modifyDBCluster` to enable the RDS Data API, then issue the extension SQL.

    Actually, the recommended CDK pattern for this is:
    1. Enable the RDS Data API on the cluster (`enableDataApi: true` on the `DatabaseCluster` construct).
    2. Add a `custom_resources.AwsCustomResource` with `onUpdate: { service: 'RDSDataService', action: 'executeStatement', parameters: { resourceArn: cluster.clusterArn, secretArn: dbSecret.secretArn, database: 'meridian', sql: 'CREATE EXTENSION IF NOT EXISTS vector;' }, physicalResourceId: cr.PhysicalResourceId.of('pgvector-init') }`.
    3. The custom resource must `DependsOn` the cluster being fully created.

    Import additions needed:
    ```ts
    import * as cr from 'aws-cdk-lib/custom-resources';
    ```

    Changes to `this.dbCluster` definition: add `enableDataApi: true` to the `DatabaseCluster` props.

    After `this.dbCluster` is defined, add:
    ```ts
    const pgvectorInit = new cr.AwsCustomResource(this, 'PgvectorInit', {
      onUpdate: {
        service: 'RDSDataService',
        action: 'executeStatement',
        parameters: {
          resourceArn: this.dbCluster.clusterArn,
          secretArn: dbSecret.secretArn,
          database: 'meridian',
          sql: 'CREATE EXTENSION IF NOT EXISTS vector;',
        },
        physicalResourceId: cr.PhysicalResourceId.of('pgvector-init'),
      },
      policy: cr.AwsCustomResourcePolicy.fromSdkCalls({
        resources: [this.dbCluster.clusterArn, dbSecret.secretArn],
      }),
    });
    pgvectorInit.node.addDependency(this.dbCluster);
    ```

    This ensures the `vector` extension is created in the `meridian` database on every CDK deploy (idempotent via `IF NOT EXISTS`).
